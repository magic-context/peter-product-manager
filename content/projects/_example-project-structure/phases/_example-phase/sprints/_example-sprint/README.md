# Sprint: [Sprint Name/Number]

This folder contains all information for a single sprint.

## Files in This Folder

- **plan.md** - Sprint goals, committed stories, capacity (created at sprint start)
- **progress.md** - Daily updates, blockers, status tracking (updated throughout)
- **user-stories/** - Detailed user story specifications

## Workflow

### Sprint Start
1. Create `plan.md` using `templates/sprint_plan.md`
2. Define sprint goal and committed stories
3. Create user story files for each committed story

### During Sprint
1. Update `progress.md` with daily standup notes
2. Track story completion, blockers, and changes
3. Add decision notes as they happen

### Sprint End
1. Complete retrospective section in `progress.md`
2. Document lessons learned
3. Archive or move incomplete stories

---

*This sprint folder is the single source of truth for this iteration.*
