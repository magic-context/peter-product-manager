# Sprints Directory

This folder contains sprints for the current phase.

## Structure

Each sprint gets its own folder:

```
sprints/
├── sprint-01/
│   ├── plan.md           # Use sprint_plan template
│   ├── progress.md       # Use sprint_details template
│   └── user-stories/     # Individual story files
├── sprint-02/
│   ├── plan.md
│   ├── progress.md
│   └── user-stories/
```

## Sprint Workflow

1. **Planning**: Create `plan.md` using the sprint_plan template
2. **During Sprint**: Update `progress.md` daily/frequently
3. **User Stories**: Add detailed story files to `user-stories/`
4. **Sprint Close**: Complete retrospective section in progress.md

## Naming

- Use `sprint-01`, `sprint-02` for sequential numbering
- Or use `sprint-auth-mvp` for descriptive names
- Be consistent within your project

---

*Each sprint folder is a complete record of that iteration.*
