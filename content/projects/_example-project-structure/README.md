# Example Project Structure

**This is a template showing the recommended project folder structure. Delete this folder when you create your first real project.**

## Folder Hierarchy

```
_example-project-structure/
├── overview.md              # ← Project overview (Level 2-3)
├── phases/                  # ← Phase breakdown (Level 4)
│   └── _example-phase/
│       ├── overview.md      # ← Phase goals and timeline
│       └── sprints/         # ← Sprint work (Level 5)
│           └── _example-sprint/
│               ├── plan.md      # ← Sprint plan
│               ├── progress.md  # ← Sprint tracking
│               └── user-stories/  # ← Story details (Level 6)
│                   └── README.md
└── documents/               # ← Supporting documents
    └── README.md
```

## How This Maps to Memory Levels

- **Level 1**: Product context (in `/product-context/`)
- **Level 2-3**: This project overview
- **Level 4**: Phases with milestones
- **Level 5**: Sprints with tasks
- **Level 6**: Individual user stories and documents

## Getting Started

1. Create your real project folder (e.g., `user-authentication/`)
2. Copy the structure shown here
3. Use templates from `/templates/` to create your files
4. Delete this `_example-project-structure/` folder
